# Cube × Math 双视角评审总结与解决方案

## 评审来源

- **视角A**：线代 + 群论 + 魔方专家（数学严谨性、概念准确性）
- **视角B**：零基础普通人（可理解性、学习获得感、产品完整度）

---

## 一、两份评审的交叉分析

两份评审在8个问题上达成了共识（专家指出的同时也是普通人遇到的），在3个问题上各自独立发现。

### 1.1 共同指出的问题（交叉验证，可信度最高）

| 问题 | 专家视角 | 普通人视角 |
|------|----------|------------|
| 第5节内容过于抽象 | "只说了中心块不变，漏了奇偶性和色向守恒" | "从第5节开始走神，全是文字，没有可以动手的演示" |
| 第6节概念堆砌 | "交换子/三明治/逆序列三个概念并列，优先级不清" | "交换子、抵消法完全陌生，缺乏直观交互" |
| Tutorial 只覆盖一步 | "角块公式是逆序列不是逆矩阵" | "只教了复原一个角块，离完整复原差太远" |
| 数学表述有误 | "置换矩阵→应为旋转矩阵"（共4处硬伤） | 普通人未直接发现，但"第4节之后开始看不懂"可能与此有关 |
| 教学闭环未完成 | "学生无法验证矩阵乘积和魔方操作之间的因果联系" | "看完整个网站还是不会还原魔方" |
| 第5章缺交互 | "InvariantDemo 只有打乱计数" | "第5节希望能点击高亮中心块" |
| 实操厅缺引导 | 未提及 | "打开实操厅完全不知道该干什么" |
| 概念边界未标注 | "向量类比暗示了向量空间结构，实际是群轨道" | 普通人未直接发现 |

### 1.2 各自独立发现

**专家独有**（普通人不易察觉但评委可能发现）：
- R 矩阵展示值 vs 代码值不一致（`[[0,1,0],[0,0,1],[1,0,0]]` vs `[[1,0,0],[0,0,-1],[0,1,0]]`）
- "26 个方块"说法不严谨（应是 20 个可动块）
- 第3章双魔方只验证了"不交换"，未验证"矩阵乘积描述了魔方操作"

**普通人独有**（专家可能忽略的用户体验）：
- "每节长得都一样，缺少视觉新鲜感"
- "第6章的路线图步骤三到七全是灰色，直接告诉我没做完"
- "需要每节末尾的'学到了什么'小结来确认理解"

## 三、分优先级解决方案

### P0：数学硬伤修正（立即修复，改动小、影响大）

#### M1：第2章"置换矩阵"概念修正

| 维度 | 内容 |
|------|------|
| 位置 | `Chapter.tsx` 第2节 keyPoints 第2条 |
| 现状 | "这种'重新排列'对应一个置换矩阵" |
| 改法 | 改为"这种'重新排列'对应一个**旋转变换矩阵**（SO(3)群中的元素）" |
| 说明文字 | 补充："R 操作的本质是把右层 8 个方块做一次绕 X 轴 90° 的刚体旋转——它的数学表示是一个三维旋转矩阵。" |

#### M2：第2章展示矩阵与代码统一

| 维度 | 内容 |
|------|------|
| 位置 | `Chapter.tsx` 第74-77行 vs `matrixMath.ts` 第28行 |
| 现状 | Chapter 展示 `[[0,1,0],[0,0,1],[1,0,0]]`，matrixMath 定义 `[[1,0,0],[0,0,-1],[0,1,0]]` |
| 改法 | 以 matrixMath.ts 为准（标准的右手定则绕X轴90°旋转），将 Chapter 展示值改为 `[[1,0,0],[0,0,-1],[0,1,0]]` |

#### M3：第5章不变量类比修正

| 维度 | 内容 |
|------|------|
| 位置 | `Chapter.tsx` 第5节 analogy |
| 现状 | "行列式 det(AB) = det(A)·det(B)，如果 A 的行列式为 0，则无法找到逆矩阵" |
| 问题 | 这是讲"可逆性"不是"不变量" |
| 改法 | 改为："**行列式是不变量的一个绝佳例子**：det(P) = ±1 恒成立，无论怎么做矩阵乘法行列式都不会变。魔方也有类似的不变量——比如中心块的相对位置永远不变，角块排列的奇偶性永远等于棱块排列的奇偶性。" |

#### M4：Tutorial 角块公式用词修正

| 维度 | 内容 |
|------|------|
| 位置 | `Tutorial.tsx` 角块介绍阶段 |
| 现状 | "这是逆矩阵组合的经典应用" |
| 改法 | 改为"这是**逆序列法则**的直接应用：(D'·R'·D·R)⁻¹ = R'·D'·R·D——把打乱步骤反过来执行。" |

### P1：教学体验升级（重点投入，影响最大）

#### T1+T3+T4：第5节和第6节增加交互演示

设计方案：

**第5节「不变量观察器」升级**：

```
现状：只有一个"随机打乱"按钮 + 计数器
目标：三模式切换
  模式1「中心块不变」(已有) —— 打乱后高亮6个中心块，文字提示"无论怎么转，6个中心块的位置始终不变"
  模式2「奇偶性检查器」（新增）—— 显示角块排列的奇偶性、棱块排列的奇偶性，
    每次打乱后自动判断"奇偶性相等 ✓"或"非法状态 ✗"
  模式3「色向校验器」（新增）—— 显示角块色向总和 mod 3、棱块色向总和 mod 2，
    始终为零，证明"合法操作不会产生非法状态"
```

**第6节「抵消法」交互升级**：

```
现状：只有 keyPoints 文字列表
目标：
  1. 新增"逆序撤销演示"——类似第4节，但增加到4-5步操作，展示逆序撤销的效果
  2. 新增"三明治变换演示"——左右两个魔方：
     左侧：只有 A 操作的效果
     右侧：P·A·P' 的效果（A"搬到"了不同位置）
     说明文字："同样的核心操作 A，通过 P·A·P' 被搬到了魔方的不同区域执行"
  3. keyPoints 改为分层展示：
     第一层「逆序撤销」(核心，对应 Tutorial 角块)：操作→逆序→复原
     第二层「三明治变换」(核心，对应 Tutorial 棱块)：P·A·P' 原理
     第三层「交换子」(高级，标注"进阶探索")：[X,Y] = X·Y·X'·Y'
```

#### T2：实操训练厅增加 AI 提示功能

| 维度 | 内容 |
|------|------|
| 现状 | 实操厅有打乱/重置/撤销/计时/模式切换，但没有引导提示 |
| 方案 | 新增「提示」按钮：点击后分析当前魔方状态，给出下一步建议操作 |
| 要求 | 至少支持"检测魔方是否已还原并给出建议"，如有能力可扩展到"识别当前阶段并给出对应步骤" |

#### M5：第3章增加"矩阵验证"环节

| 维度 | 内容 |
|------|------|
| 现状 | 只验证了"顺序不同结果不同"，没验证"矩阵乘积正确描述了魔方操作" |
| 方案 | 在 ComposeVerify 演示完成后，增加一行醒目文字：<br>"验证：R 操作的矩阵 × U 操作的矩阵 = [[...],[...],[...]]<br>将角块 (1,1,1) 的坐标代入计算：旋转前 = (1,1,1)，旋转后 = (?,?,?)<br>观察右侧魔方中对应角块的实际位置——与计算结果完全一致。" |

#### M6：第5章增加奇偶性不变量的文字说明

| 维度 | 内容 |
|------|------|
| 现状 | 只提到"中心块不变"一个不变量 |
| 方案 | 在 keyPoints 增加一条：<br>"角块排列的奇偶性和棱块排列的奇偶性在任意合法操作下始终相同——这就是'不可能只交换两个角块而其他一切不变'的数学原因。" |

### P2：细节打磨（后续迭代，不影响核心评审）

#### M7+M8：概念表述精修

- 第1章 keyPoints："26 个方块" → "魔方有 26 个可见方块，其中 6 个中心块固定不动，实际可动的是 8 个角块 + 12 个棱块 = 20 个。"
- 在 Archive（数学档案馆）中增加一节"类比边界声明"，说明"魔方状态空间本质是群作用的轨道，不是向量空间。'向量'和'矩阵'是入门类比，不是严格数学定义。"

#### T5：Tutorial 步骤扩展

- 当前：白色十字（观察） → 白色角块（3子阶段） → 中层棱块（3子阶段）
- 优先扩展：黄色面公式（OLL 基础版）——这是层先法的第4步，补完后用户能看到"我真的在学会还原"
- 后续扩展：顶层棱角归位（PLL 基础版）——至此完成六面还原

#### T6：每节末尾增加小结

格式示例（第2节小结）：

> 本节要点
> 1. 魔方的每一次转动，都可以用一个 3×3 旋转矩阵来表示
> 2. 多次连续转动 = 矩阵连乘
> 3. 矩阵乘法不满足交换律——先R后U ≠ 先U后R（下一节演示）

#### T7：中层棱块公式说明精修

将"三明治变换（共轭）"改为"这是一个组合操作，前后两段各使用一次三明治变换：前半段腾出空间，后半段塞入棱块。"

#### T8：章节视觉差异化

- 每节在右侧魔方上方增加该节的核心视觉标识（不同颜色/图标）
- 或者每节的卡片背景色微调（浅色底纹区分）

## 四、实施建议

### 4.1 分三个批次推进

| 批次 | 内容 | 预计工作量 | 目标 |
|------|------|------------|------|
| 第一批（评审前必须） | P0 全部（M1-M4）：4处数学硬伤修正 | 1-2小时 | 消除数学评委可能发现的"硬伤" |
| 第二批（大幅加分） | P1 中 T1+T3+T4（第5/6节交互升级）+ M5（矩阵验证）+ M6（不变量补充） | 1-2天 | 解决两方评审共同指出的核心问题 |
| 第三批（锦上添花） | P1 中 T2（AI提示）+ P2 全部细节打磨 | 1-3天 | 补全产品体验闭环 |

### 4.2 最低完成标准

如果时间紧迫，至少完成**第一批全部 + 第二批中的第5节交互升级**，原因：

1. P0 数学硬伤是评委能直接看出来的，不修会扣分
2. 第5节是两份评审交叉验证的"用户流失点"——修这一处能同时解决专家和普通人的共同痛点
3. 这些改动以文字修改为主，技术实现简单，风险低

## 五、两份评审的核心共识

两份评审虽然视角完全不同，但在最高层面达成了一个共识：

**这个产品的创意方向是对的——"用数学教魔方"切中了真正的教育空白。但当前 Demo 的数学表述存在若干不准之处，教学闭环没有完成，导致既有数学背景的人能看出硬伤，又没有数学背景的人学不会。修完数学硬伤 + 补上交互演示 + 推进教学步骤覆盖，这个产品才能真正立住。**
